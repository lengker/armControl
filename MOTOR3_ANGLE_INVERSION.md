# 电机3角度取反说明

## 问题发现

测试时发现：
- 发送 `set 3 10`（正角度）→ 小臂**下压**
- 发送 `set 3 -20`（负角度）→ 小臂**上抬**

这与代码中的数学定义相反。

## 根本原因

**电机3的安装方向与数学定义相反**：

### 数学定义（FK/IK计算）
- theta2 = 0° → 小臂水平
- theta2 > 0° → 小臂向上
- theta2 < 0° → 小臂向下

### 电机实际（硬件）
- 发送 0° → 小臂水平
- 发送 +10° → 小臂向下
- 发送 -10° → 小臂向上

## 解决方案

**在电机控制层取反**：
- FK/IK计算仍使用数学定义（正=向上）
- 发送给电机时取反：`set_angle(ID_SMALL, -theta2, ...)`
- 这样保持了数学一致性，只在硬件接口层处理差异

## 修改位置

所有 `set_angle(ID_SMALL, ...)` 调用都已修改为 `set_angle(ID_SMALL, -theta2, ...)`：

1. `Motion_SetHome()` - 回零
2. `Motion_MoveToXYZ()` - 普通运动
3. `Motion_MoveToXYZ_Smooth()` - 平滑运动
4. `Motion_MoveToXYZ_RuckigSmooth()` - Ruckig平滑运动
5. `Motion_TestSingleMotor()` - 单电机测试

## 验证方法

### 测试1：单电机控制
```
mode test
set 3 30      # 应该看到小臂向上30度
set 3 -30     # 应该看到小臂向下30度
set 3 0       # 应该回到水平
```

### 测试2：FK正解验证
```
mode test
set 2 70      # 大臂70度
set 3 10      # 小臂10度（向上）
```

观察FK打印：
- `Th2=10.0` （数学角度，正值）
- `L2 绝对角=10.0` （应该向上）
- 实际小臂应该向上抬起

### 测试3：完整运动
切换到自动模式，让机械臂抓取物体，观察：
- 小臂运动方向是否正确
- 末端位置是否准确
- 是否还有卡住现象

## 注意事项

1. **current_joint_2 保存的是数学角度**（未取反）
   - 用于FK计算
   - 用于角度限制检查

2. **发送给电机的是取反后的角度**
   - 只在 `set_angle()` 调用时取反
   - 不影响其他计算

3. **测试模式打印**
   - 会同时显示数学角度和电机角度
   - 例如：`数学角度30.0°, 电机角度-30.0°`

## 对系统的影响

### 之前（错误）
- IK计算：需要theta2=30°（向上）
- 发送给电机：30°
- 电机实际：向下30°
- 结果：**位置错误**

### 现在（正确）
- IK计算：需要theta2=30°（向上）
- 发送给电机：-30°
- 电机实际：向上30°
- 结果：**位置正确**

## 总结

这个修改确保了：
1. ✅ FK/IK计算使用统一的数学定义
2. ✅ 电机实际运动与数学定义一致
3. ✅ 末端位置计算准确
4. ✅ 不会因为角度方向错误导致卡住

**这是一个关键修复，直接影响机械臂的运动准确性！**
