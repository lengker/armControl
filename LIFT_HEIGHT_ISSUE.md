# 抬升高度问题说明

## 问题现象

抬升时IK失败：
```
目标: X=24.1, Y=-433.3, Z=-330.6  // 箱子位置
→ 抬升箱子到 Z=299.4              // 目标高度
[Ruckig] IK failed                 // 失败！
```

## 根本原因

### 之前的逻辑（错误）
```c
float lift_z = grab_z + PICK_LIFT_OFFSET_Z_MM;
// grab_z = -330.6mm
// PICK_LIFT_OFFSET_Z_MM = 630mm
// lift_z = -330.6 + 630 = 299.4mm
```

**问题**：
1. 箱子位置Z=-330.6mm（很低）
2. 加上630mm的偏移，目标Z=299.4mm
3. 从Z=-330.6到Z=299.4，跨度630mm
4. 在XY位置不变的情况下，这个高度**超出了机械臂的工作空间**
5. IK无解，运动失败

### 为什么会超出工作空间？

机械臂的工作空间是有限的：
- 大臂长度：350mm
- 小臂长度：235.5mm
- 最大伸展：约585mm

当箱子在很远的位置（Y=-433mm）且很低（Z=-330mm）时：
- 机械臂已经接近最大伸展
- 再抬升630mm，需要收回机械臂
- 但XY位置不变，机械臂无法同时满足XY和Z的要求
- 导致IK无解

## 解决方案

### 改为使用绝对高度

```c
float lift_z = PICK_SAFE_LIFT_HEIGHT_MM;  // 100mm
```

**优点**：
1. 不管箱子在哪里，都抬升到固定的安全高度
2. 避免超出工作空间
3. 更可预测，更安全

**配置**：
```c
#define PICK_SAFE_LIFT_HEIGHT_MM    100.0f   // 安全抬升高度
```

### 如何选择安全高度？

**考虑因素**：
1. **足够高**：避免碰到地面障碍物
2. **不太高**：避免超出工作空间
3. **可达性**：在所有可能的XY位置都能到达

**推荐值**：
- Z=50mm：最低安全高度
- Z=100mm：推荐值（平衡安全性和可达性）
- Z=150mm：较高，但某些远距离位置可能无法到达

### 测试方法

1. **测试不同箱子位置**：
   ```
   箱子1: X=100, Y=-300, Z=-200  → 抬升到Z=100
   箱子2: X=200, Y=-400, Z=-350  → 抬升到Z=100
   箱子3: X=50,  Y=-500, Z=-100  → 抬升到Z=100
   ```

2. **观察是否成功**：
   - IK是否有解
   - 机械臂是否卡住
   - 抬升高度是否足够

3. **调整安全高度**：
   - 如果经常IK失败 → 降低安全高度
   - 如果抬升不够高 → 提高安全高度（但要确保IK有解）

## 其他注意事项

### 预备点高度

预备点使用相对偏移是合理的：
```c
float approach_z = box_z + PICK_APPROACH_OFFSET_Z_MM;
```

因为：
1. 预备点在箱子正上方
2. 偏移量较小（150mm）
3. 不会超出工作空间

### 放置点高度

放置点使用固定坐标：
```c
#define PLACE_POINT_Z_MM  200.0f
```

这也是合理的，因为放置区位置是固定的。

## 总结

| 位置 | 高度计算方式 | 原因 |
|------|------------|------|
| 预备点 | `box_z + 150mm` | 相对偏移，偏移量小 |
| 抓取点 | `box_z` | 箱子实际位置 |
| 抬升点 | `100mm`（绝对高度） | 避免超出工作空间 |
| 放置点 | `200mm`（固定坐标） | 放置区固定 |

**关键原则**：
- ✅ 小偏移可以用相对高度
- ✅ 大跨度必须用绝对高度
- ✅ 固定位置用固定坐标
- ❌ 避免大偏移+相对高度的组合
