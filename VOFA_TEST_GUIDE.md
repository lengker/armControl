# VOFA上位机测试指南

## 系统概述

本文档说明如何在VOFA上位机软件中实现：
- **滑块控制**：X/Y/Z坐标（实时调整机械臂末端位置）
- **按钮控制**：气泵开/关（模拟抓取动作）
- **实时反馈**：查看当前气泵状态和机械臂状态

---

## 硬件连接

```
STM32G474
  ├─ USART1 (PC4/PC5) ──┬─→ USB转串口模块 ──→ 电脑 (COM口)
  │                      └─→ VOFA上位机
  │
  ├─ FDCAN1 (PA11/PA12) ──→ CAN总线 ──→ 大然电机驱动器
  │
  ├─ GPIO PB10 ──→ 气泵 (Pump)
  │
  └─ TIM1 (PC0) ──→ 舵机 PWM
```

## VOFA协议 (UART1 @ 115200 baud)

### 帧格式
```
[帧头] [命令] [数据0] [数据1] [数据2] [数据3-4]
 0xFF  0xXX    Byte0   Byte1   Byte2   Byte3-4

总长度：7字节
字节序：小端 (LSB first)
```

### 命令集

| 命令 | Code | 参数 | 说明 |
|------|------|------|------|
| 设置X坐标 | 0x10 | float (Bytes 0-3) | 目标X坐标(mm) |
| 设置Y坐标 | 0x11 | float (Bytes 0-3) | 目标Y坐标(mm) |
| 设置Z坐标 | 0x12 | float (Bytes 0-3) | 目标Z坐标(mm) |
| 控制气泵 | 0x20 | uint8 (Byte 0) | 0=关, 1=开 |
| 查询泵状态 | 0x21 | 无 | 返回当前状态 |

### 示例CAN消息

#### 设置 X = 350.0mm
```
Frame (16进制):
  FF 10 00 00 B0 43
  ↑  ↑  └─────┬─────┘
  │  │        └─ 350.0 (float)
  │  └─ 命令: 0x10
  └─ 帧头

转换：0x43B00000 = float(350.0)
```

#### 开启气泵
```
Frame (16进制):
  FF 20 01 00 00 00
  ↑  ↑  ↑
  │  │  └─ 数据: 1 (开启)
  │  └─ 命令: 0x20
  └─ 帧头
```

#### 关闭气泵
```
Frame (16进制):
  FF 20 00 00 00 00
  ↑  ↑  ↑
  │  │  └─ 数据: 0 (关闭)
  │  └─ 命令: 0x20
  └─ 帧头
```

## VOFA配置步骤

### 1. 下载和安装VOFA

查阅官方网站：https://www.vofa.plus/

### 2. 新建项目

①. 打开VOFA ++ 
② "File" → "New Project"
③ 选择"发送固定的数据包"模式
④ 设置串口参数：
   - COM口：选择STM32的USB串口
   - 波特率：115200
   - 数据位：8
   - 停止位：1
   - 校验位：无

### 3. 添加控件

#### 添加3个滑块控制X/Y/Z

**控件1 - X坐标滑块**

```
名称：设置X坐标
类型：滑块 (Slider)
范围：0 ~ 500 (mm)
步长：1

输出格式：发送原始数据包
数据包内容(16进制)：
  FF 10 [Byte0] [Byte1] [Byte2] [Byte3]
  
其中 [Byte0-3] 为X值的float字节表达式
```

**在VOFA中配置数据包的方法**：
1. 右键滑块 → "绑定数据"
2. 选"浮点型(Float)" → 小端法
3. 输入变量名：`target_x`
4. 获取数据包模式：勾选"前缀"
5. 前缀填写：`FF 10`
6. 后缀：`00 00`（可选）

同理配置Y/Z滑块，分别用命令码 `0x11` / `0x12`。

#### 添加1个按钮控制气泵

**控件4 - 气泵切换按钮**

```
名称：吸取 (按下开启, 松开关闭)
类型：按钮 (Button)
按下状态：发送数据包 [FF 20 01 00 00 00] (开启)
松开状态：发送数据包 [FF 20 00 00 00 00] (关闭)
```

或者两个单独的按钮：

```
按钮A - "吸取"
  点击发送：[FF 20 01 00 00 00]
  
按钮B - "放置"
  点击发送：[FF 20 00 00 00 00]
```

#### 添加文本框显示反馈

```
名称：气泵状态
类型：文本区域
绑定数据：串口返回的 [PUMP_STATE] 后面的数值
```

## VOFA Python脚本示例（高级用法）

如果VOFA支持Python脚本，可编写自动化测试脚本：

```python
import time

# 设置坐标并抓取
def grab_at_position(x, y, z):
    # 1. 设置目标坐标
    send_vofa_command(0x10, x)  # X
    time.sleep(0.1)
    send_vofa_command(0x11, y)  # Y
    time.sleep(0.1)
    send_vofa_command(0x12, z)  # Z
    time.sleep(0.5)  # 等待机械臂到位
    
    # 2. 开启气泵
    send_vofa_command(0x20, 1)  # Pump ON
    time.sleep(2.0)
    
    # 3. 关闭气泵
    send_vofa_command(0x20, 0)  # Pump OFF

# 测试循环
grab_at_position(300, 0, 200)   # 第1个抓取点
time.sleep(1.0)
grab_at_position(350, 50, 150)  # 第2个抓取点
```

## 现场测试流程

### Phase 1: 基础通信测试
1. 编译烧写固件
2. 打开VOFA，连接COM口
3. 拖动X滑块，观察目标坐标变化是否反映到串口日志
4. 预期输出：`target_x updated to 350.000000`

### Phase 2: 气泵控制测试
1. 点击VOFA中的"吸取"按钮
2. 观察：
   - 串口输出 `[PUMP] ON`
   - 实际气泵发出工作声
3. 点击"放置"按钮
4. 观察：
   - 串口输出 `[PUMP] OFF`
   - 气泵停止工作

### Phase 3: 姿态稳定测试
1. 在VOFA设置 X=300mm, Y=0mm, Z=200mm
2. 机械臂运动到该位置
3. **轻轻摇晃机械狗**，观察机械臂末端的稳定性
4. 串口输出应该显示：
   ```
   [Stab-Wrist] Roll=2.15°(→-0.10°), Pitch(Wrist)=-1.23°(→-0.08°)
   ```

### Phase 4: 完整抓取任务测试
1. 在VOFA设置第一个位置 (X=300, Y=0, Z=250)
2. 等待机械臂到位
3. 点击"吸取"按钮，持续2秒
4. 改变坐标到目标位置
5. 点击"放置"按钮关闭气泵
6. 验证物体是否正确交付

## 故障排查

| 症状 | 原因 | 解决方案 |
|------|------|---------|
| VOFA无法连接 | COM口错误或驱动缺失 | 检查设备管理器，确认USB转串口正常 |
| 收不到反馈 | 波特率不匹配 | 确保设为115200 baud |
| 发送命令无反应 | 串口格式错误 | 检查帧格式是否为[FF][CMD][Data...] |
| 气泵不工作 | GPIO/硬件问题 | 检查pump.c的Pump_On/Off函数 |
| 坐标变化但机械臂不动 | 运动学解算失败 | 检查逆运动学范围，增加Z值（增加可达性） |
